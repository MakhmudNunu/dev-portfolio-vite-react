# Developer Portfolio Frontend (React + Vite)

Фронтенд-часть портфолио на базе **React 19** и **Vite**, созданная специально для интеграции с готовым бэкендом на **FastAPI**.

> Этот репозиторий содержит только клиентскую часть. Все данные формы обратной связи и метрики получаются из существующего backend API, развёрнутого на Render.

## Что делает этот фронтенд

- Рендерит посадочную страницу портфолио с секциями: «Обо мне», «Услуги», «Проекты» и «Контакты».
- Поддерживает отправку формы обратной связи на FastAPI backend через `POST /api/contact/`.
- Получает и отображает метрики из `GET /api/metrics`.
- Обрабатывает статус загрузки, ошибки валидации и rate limit.
- Показывает ответ AI от бекенда в красивой плашке после успешной отправки.

## Как запустить

### 1. Установите зависимости

```bash
npm install
```

### 2. Настройте переменные окружения

Создайте файл `.env` в корне проекта и укажите URL вашего FastAPI-бэкенда:

```env
VITE_API_BASE=https://your-backend-on-render.onrender.com
```

> Если переменная не задана, используется заглушка `https://твой-бэкенд-на-render.onrender.com`.

### 3. Запустите dev-сервер

```bash
npm run dev
```

Откройте приложение в браузере по адресу, который выведет Vite.

## API интеграция

Фронтенд ожидает, что backend уже работает и предоставляет следующие эндпоинты:

- `POST /api/contact/`
  - Тело запроса: `{ name: string, email: string, phone: string, comment: string }`
  - Успешный ответ: `201 Created` с JSON `{ success: true, message: string, data: { id: number, name: string }, ai_reply: string }`
- `GET /api/metrics`
  - Ответ: `{ total_submissions: number, ai_sentiment_breakdown: { positive: number, neutral: number, negative: number } }`

### Что делает фронтенд

- Собирает данные формы из полей: ФИО, Email, Телефон, Сообщение.
- Блокирует кнопку и показывает состояние `Отправка...` во время запроса.
- Обрабатывает ошибки:
  - `422` — ошибка валидации данных.
  - `429` — rate limit.
  - другие статусы — отображает текст ошибки.
- После `201 Created` показывает ответ AI (`ai_reply`) и обновляет метрики.
- При загрузке страницы запрашивает метрики и показывает их в карточках.

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `VITE_API_BASE` | Базовый URL FastAPI backend. Например `https://your-backend-on-render.onrender.com` |

## Прокси и CORS

Если frontend и backend работают на разных доменах, убедитесь, что FastAPI разрешает CORS-запросы с origin вашего фронтенда.

> Вариант для локальной разработки: в `vite.config.ts` можно добавить прокси для `/api` на backend.

## Почему этот фронтенд создан

Этот клиентский проект был специально написан для работы с готовым FastAPI-бэкендом. Фронтенд не содержит собственного API, а полностью опирается на существующие backend-эндпоинты для отправки формы и вывода метрик.
